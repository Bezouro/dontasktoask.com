Não peça para perguntar, Apenas pergunte

As vezes, nas salas de chat em que participo, alguem aparece e pergunta algo como

Foobar123:

Alguém entende de Java?

Isto está errado, por varias razões. O que ele está realmente parguntando é,

Foobar123:

Algum especialista em Java que esteja disposto a se comprometer em ver meu problema, seja lá o que for, mesmo que não seja relacionado a Java ou se alguém que não sabe nada sobre Java poderia realmente responder à minha pergunta?

Existem muitas razões pelas quais as pessoas que veem sua pergunta não a responderiam. Ao perguntar, você está pedindo mais do que acredita que está pedindo.

Você está pedindo às pessoas que assumam a responsabilidade. Você está questionando a confiança que as pessoas tem em suas proprias habilidades. Você também está deixando de fora desnecessariamente as outras pessoas. Costumo responder a perguntas relacionadas a linguagens ou bibliotecas que nunca usei, porque as respostas são de senso comum (para um programador).

Isto também pode ser visto como...

Foobar123:

tenho uma pregunta sobre Java mas tenho preguiça de colocar em palavras, a menos que haja alguém capaz de responde-la neste canal.

..o que é vago. Se você não está disposto em fazer algum trabalho para resolver seu problema, Por que outros deveriam?

A solução não é pedir para perguntar, apenas pergunte. Alguém que está ausente no chat e de vez em quando olha o que está acontecendo não vai querer responder à sua pergunta "pedindo para perguntar", mas se você vir a descrição do seu problema pode despertar interesse e levá-los a responder.

Então, para resumir não pergunte: "há um especialista em Java por aqui?", ao invés disso, pergunte: "como faço para resolver [um problema] com Java e [outras informações relevantes]?"

Outros problemas similares: O problema XY, Não diga apenas Oi. Para mais sobre este tema: Como fazer uma boa pergunta?, ou se tiver mais tempo: Como fazer preguntas inteligentes.
